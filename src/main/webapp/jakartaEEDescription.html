Jakarta EE

Jakarta EE, ранее известный как Java EE (Java Enterprise Edition) и J2EE (Java 2 Enterprise Edition), представляет собой набор спецификаций, разработанных для создания крупномасштабных многопользовательских систем уровня предприятия. Эти спецификации включают в себя технологии и стандарты, такие как Servlets, JSP, EJB и многие другие, которые обеспечивают функциональность для разработки корпоративных приложений.

После того как Oracle передал управление Java EE в Eclipse Foundation в 2017 году, название было изменено на Jakarta EE. Это изменение было продиктовано желанием обеспечить более открытый и сообщественно-ориентированный подход к развитию платформы.

Jakarta EE широко используется в индустрии для создания масштабируемых, надежных и безопасных веб-приложений и сервисов. Основное преимущество использования Jakarta EE заключается в том, что разработчики могут сконцентрироваться на написании бизнес-логики, пока сама платформа заботится о многих аспектах инфраструктуры, таких как управление транзакциями, безопасность и управление жизненным циклом компонентов.

Основные особенности и компоненты Jakarta EE:


Servlets и JavaServer Pages (JSP): Сервлеты - это серверные программы, которые принимают запросы от клиентов, обрабатывают их и отправляют ответ обратно клиенту. JSP, с другой стороны, предоставляют способ создания динамического содержания на основе HTML или XML.


Enterprise JavaBeans (EJB): EJB предоставляет архитектуру для разработки масштабируемых, долгоживущих и безопасных приложений на основе Java.


Java Persistence API (JPA): Это спецификация, которая предоставляет механизм для сохранения объектов Java в базе данных.


JavaServer Faces (JSF): JSF - это структура MVC (Model-View-Controller) для создания компонентно-ориентированных пользовательских интерфейсов для веб-приложений.

Java Message Service (JMS): JMS предоставляет механизм для асинхронной коммуникации через сообщения в Java.


Contexts and Dependency Injection (CDI): CDI предоставляет мощные функции управления жизненным циклом объектов и инъекции зависимостей.

Микросервисы и облачные возможности: С Jakarta EE можно создавать модульные и масштабируемые микросервисы, готовые к работе в облачной среде.
